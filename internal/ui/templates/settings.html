Varnish Cache

Accelerate static assets and cache-eligible endpoints through a local Varnish daemon. Traffic is WAF-inspected first, then handed to Varnish with spoofable host headers scrubbed; cache misses return to the WAF, which routes them to the right backend automatically, so the Varnish config never needs editing when services change. Just enable caching per service on the Services page. Varnish is set up by the installer (or see deploy/varnish/README.md).
